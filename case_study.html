THE CHALLENGE

Forty minutes of useful work, twenty minutes of writing it up.

Across a typical Deloitte Greece engagement, a Manager attends 6–10 client and internal meetings per week. Each call generates 20–40 minutes of post-work — re-listening, extracting decisions, attributing actions, drafting a polished MoM, and emailing it before the day ends. The deliverable is high-stakes; clients judge governance maturity by what lands in their inbox. Yet the process is repetitive, error-prone, and almost universally rushed.

⌛

Generic LLMs miss the structure

ChatGPT can summarise but produces inconsistent shape. Action items lose owners, decisions get buried, formatting drifts between runs.

⚙

Native tools can't enforce house style

Microsoft Copilot Recap is fast but cannot encode a team's writing standard. Your house style needs custom rules — and there's no way to plug them in.

⌖

Transcription tools stop short

Otter, Fireflies and similar produce a transcript and a thin summary — not a structured Word document with action-item table, risks, and owners ready to send.

THE SOLUTION

One Word file holds the rules. The AI follows them. You stay in control.

A Streamlit web UI ingests any Teams transcript export, applies a team-owned writing standard from Rules.docx, and produces a styled Word document plus an Outlook draft email. Six pillars set this apart from off-the-shelf alternatives.

01

Drop in. One click out.

Drop a Teams export (.docx / .txt / .pdf / .vtt). One click produces a styled Word document, a transcript audit trail, and an Outlook draft email — under thirty seconds.

02

House-style enforced via Rules.docx

An editable Word document holds the team's writing standard. The AI reads it on every run. Edit the rules in Word — the next MoM follows them automatically. No prompt engineering, no retraining.

03

Bilingual by design

Greek and English transcripts are parsed natively. Vocative case is resolved (Νίκο → Νικόλας Βασιλείου). Code-switched English business terms are preserved. Greek discourse fillers stripped alongside English ones.

04

Two-tier quality scoring

Every generated MoM is scored on Structure / Hygiene / Send-readiness. With an API key, a second pass uses Claude as a senior-PM judge — Tone, Completeness, Accuracy, Structure on a 1–10 rubric, plus a written critique.

05

Continuous learning loop

Every piece of feedback can become a permanent rule. Users rate the output, optionally describe what should change, and Claude proposes a new rule in the existing tone. One click writes it back to Rules.docx.

06

Manual override at every stage

An inline editor lets the user adjust any field before exporting — meeting metadata, executive summary, every action item, every risk. Edits regenerate the Word document on the spot.

HOW IT WORKS

Twelve stages — three loops. The third one closes back on itself.

The first nine stages run on every meeting. The last three stages — feedback, rule proposal, rule application — happen continuously and lift baseline quality with use.

KEY INNOVATIONS

Six things that make this different from a generic LLM wrapper.

01 Rules.docx as a living spec

Most LLM apps hide their prompt. Ours puts it in a Word document the user owns and edits. Two sections — Step 1 (transcript cleanup) and Step 2 (writing rules) — define everything from filler-word stripping to action-item phrasing. Style is editable, auditable, version-able.

02 Continuous learning without retraining

User submits a comment. Claude proposes a new rule in the same tone as the existing 30+. User clicks Apply. The rule appends under "Step 3 — Continuous improvements" in Rules.docx. The next MoM follows it. No fine-tuning. No deployment. A Word document grows smarter.

03 Side-by-side AI corrections audit

A toggle shows the original transcript turn-by-turn next to the cleaned version, with red strike-through on every removed filler and green highlight on additions. Reviewers and auditors verify exactly what the AI changed before trusting the deliverable.

04 Greek vocative resolution

Greek meetings address people in vocative case (Νίκο, Δημήτρη). When the transcript says "Δημήτρη, παρακαλώ escalate-αρε στον Ανδρέα", the AI correctly attributes the action to Δημήτρης Αντωνίου — the full nominative name from the participant list.

05 Materiality filter

Senior clients hate noise. Acknowledgments ("Will do", "Got it"), agenda transitions, logistical micro-tasks ("send a calendar hold") get filtered out before the MoM is rendered. The deliverable reads like a curated executive brief, not a transcript dump.

06 Two-layer quality assurance

Deterministic scoring runs at zero cost on every generation (Structure 40 + Hygiene 30 + Send-ready 30). When budget allows, Claude grades as a senior PM would — Tone, Completeness, Accuracy, Structure on a 1–10 rubric, plus a written critique.

SECURITY & TRUST

The questions every Deloitte buyer asks. Answered up front.

The accelerator is designed for use inside the Deloitte network with client-confidential transcripts. Four design principles keep it safe.

🔒

Local-first by default

The transcript is read on your machine and processed in-memory. The only outbound call is to the Anthropic API for the cleaned text — no audio, no original file, no metadata. Without an API key, the offline rule-based fallback runs entirely on-device.

📜

Auditable transformations

Every cleanup step shows a side-by-side diff (red strike-through on removals, green on additions). The full Rules.docx is open Word — readable by reviewers, editable by the team, traceable in version control. No hidden prompts.

🛑

No fine-tuning, no retention

The pipeline never trains on your data. Anthropic's enterprise API does not retain inputs. The continuous-learning loop is implemented as Word-document amendments — not model weights — so improvements are inspectable and reversible.

📊

Cost transparency

Every Claude call has an explicit token count surfaced in the logs. Per-meeting cost is bounded under €0.02 (Sonnet). The Quality-check feature is opt-in and capped. No surprise bills, no token amplification, no autoplay loops.

✓

Deployment posture: The tool runs on the consultant's laptop. The Word output stays in /output on the consultant's machine. The transcript audit trail (JSON sidecar) stays alongside it. Nothing is shipped to a shared cloud unless the consultant explicitly attaches the Word doc to an email — which is the same workflow they had before.
